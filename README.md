<h1>RememberMe - Gestão de Atividades e Roteiros</h1>

<h4>🚀 MVP da SPRINT em Gestão de Projetos e Produtos na PUC-Rio</h4>

<h2>📌 Sobre o projeto</h2>

<p>
  O RememberMe é um app mobile com o objetivo de auxiliar usuários no gerenciamento
  de atividades do dia a dia, como compromissos, eventos e tarefas importantes.
</p>



<p>
  A aplicação permite que o usuário cadastre, visualize, edite, exclua e filtre atividades,
  além de possibilitar a criação de roteiros associados a eventos ou compromissos que precisam
  seguir uma sequência de etapas.
</p>

<p>
  O projeto de gestão foi estruturado com  histórias de usuário, critérios de aceite,
  Definition of Ready (DoR), Definition of Done (DoD), backlog e organização por sprints.
</p>

 
<h2>🎯 Objetivo do MVP</h2>

<p>
  O objetivo inicial do MVP é entregar as principais funcionalidades necessárias para que
  o usuário consiga se cadastrar, acessar o sistema e gerenciar suas atividades.
</p>

<p>
  As funcionalidades principais incluem:
</p>

<ul>
  <li>Cadastro de usuário</li>
  <li>Login de usuário</li>
  <li>Cadastro de atividades</li>
  <li>Visualização de atividades</li>
  <li>Edição de atividades</li>
  <li>Exclusão de atividades</li>
  <li>Filtro de atividades</li>
  <li>Visualização de atividades próximas de acontecer</li>
</ul>

<h2>✅ Definition of Ready</h2>

<p>Para uma história ser considerada pronta para desenvolvimento, ela deve atender aos seguintes critérios:</p>

<ul>
  <li>A história deve estar refinada</li>
  <li>O ambiente de desenvolvimento deve estar pronto e devs com acesso</li>
  <li>O ambiente de homologação deve estar pronto</li>
</ul>

<h2>✅ Definition of Done</h2>

<p>Para uma entrega ser considerada concluída, ela deve atender aos seguintes critérios:</p>

<ul>
  <li>Testes end-to-end realizados</li>
  <li>Pipeline de CI/CD validado</li>
  <li>Critérios de aceite atendidos</li>
  <li>Código revisado</li>
</ul>
